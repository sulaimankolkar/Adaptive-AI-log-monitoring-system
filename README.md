# Adaptive AI Monitoring Platform

A comprehensive web application for monitoring data drift in machine learning models with AI-powered insights and guardrail validation.

## Features

- **Dashboard**: Real-time overview of drift analysis results, AI insights, and guardrail logs
- **Datasets**: Upload and manage baseline and production datasets for drift analysis
- **Drift Analysis**: Run statistical drift detection tests comparing dataset distributions
- **AI Insights**: View AI-generated insights and recommendations for drift analysis results
- **Prompt Engineering**: Customize prompt templates used for AI insight generation
- **Reports**: Generate and download PDF reports of drift analysis results

## Tech Stack

### Frontend
- React 18 with TypeScript
- Material-UI (MUI) v5 for UI components
- Vite for build tooling
- React Router for navigation
- Recharts for data visualization

### Backend
- FastAPI for REST API
- SQLAlchemy for database ORM
- PostgreSQL for data storage
- OpenAI API for AI-powered insights
- ReportLab for PDF generation

## Getting Started

### Prerequisites
- Node.js 18+ and npm
- Python 3.10+
- PostgreSQL database

### Backend Setup

1. Navigate to the backend directory:
```bash
cd backend
```

2. Create a virtual environment:
```bash
python -m venv .venv
```

3. Activate the virtual environment:
```bash
# Windows
.venv\Scripts\Activate.ps1

# Linux/Mac
source .venv/bin/activate
```

4. Install dependencies:
```bash
pip install -r requirements.txt
```

5. Set up environment variables (create `.env` file):
```env
DATABASE_URL=postgresql://user:password@localhost/dbname
SECRET_KEY=your-secret-key
OPENAI_API_KEY=your-openai-api-key
```

6. Initialize the database:
```bash
python -c "from app.db.init_db import init_db; from app.core.database import SessionLocal; init_db(SessionLocal())"
```

7. Start the backend server:
```bash
uvicorn app.main:app --reload
```

The backend will run on `http://127.0.0.1:8000`

### Frontend Setup

1. Navigate to the frontend directory:
```bash
cd frontend
```

2. Install dependencies:
```bash
npm install
```

3. Start the development server:
```bash
npm run dev
```

The frontend will run on `http://localhost:5173`

### Building for Production

```bash
npm run build
```

## Usage Guide

### Authentication

The application requires authentication to access. Use the following test credentials:

- **Email**: `admin@example.com`
- **Password**: `adminpassword`

These credentials are pre-configured in the database initialization script.

### Pages Overview

#### Dashboard
The main overview page displaying:
- **Key Performance Indicators**: System status, feature drift percentage, critical features count, and AI guardrail checks passed
- **Feature Drift Chart**: Visual representation of Population Stability Index (PSI) across monitored features
- **AI Explainable Insights**: Executive summary, key findings, and actionable recommendations generated by AI
- **Guardrail Audit Log**: Verification results from AI safety middleware during prompt execution

**How to use**: Select a drift analysis job from the dropdown to view detailed metrics. The dashboard updates in real-time as new analyses are completed.

#### Datasets
Manage your data assets for drift analysis:
- Upload new datasets in CSV format
- View all datasets with version history
- Track row counts and column counts
- Monitor creation timestamps

**How to use**: Click "Upload Dataset" to add a new CSV file. Provide a name and description, then select the file. The system automatically creates version snapshots for tracking.

#### Drift Analysis
Run statistical drift detection tests:
- Select reference (baseline) dataset and version
- Select target (production) dataset and version
- Trigger drift analysis pipeline
- View historical analysis runs with status tracking
- Download reports for completed analyses

**How to use**: Choose your reference and target datasets from the dropdowns, then click "Run Statistical Pipeline". The analysis will compare distributions using Kolmogorov-Smirnov and Chi-Square tests.

#### AI Insights
View AI-generated insights for drift analyses:
- Select a drift analysis from the dropdown
- View detailed insight metadata (severity, model used, generation time)
- Read executive summary and key findings
- Review actionable recommendations
- Check guardrail validation results
- Generate new insights if not already available

**How to use**: Select a completed drift analysis job to view its AI insights. If insights haven't been generated, click "Generate Insights" to trigger the AI analysis.

#### Prompt Engineering
Customize prompt templates for AI insight generation:
- Select an existing prompt template
- Edit system instructions (behavior control)
- Modify prompt structure body with placeholders
- View prompt execution logs
- Test prompt rendering with sample data

**How to use**: Select a template from the dropdown to edit it. Use placeholders like `{ref_version}`, `{target_version}`, and `{drift_metrics}` to inject runtime data. Save changes to update future AI insights.

#### Reports
Generate and download PDF reports:
- View all generated drift analysis reports
- Generate new reports for completed analyses
- Download PDF reports for offline viewing

**How to use**: Click "Generate Report" for any completed drift analysis to create a PDF summary. The report includes metrics, charts, and AI insights.

## API Endpoints

### Authentication
- `POST /api/v1/auth/login` - User login
- `POST /api/v1/auth/register` - User registration

### Datasets
- `GET /api/v1/datasets/` - List all datasets
- `POST /api/v1/datasets/` - Create new dataset
- `POST /api/v1/datasets/{id}/upload` - Upload dataset file

### Drift Analysis
- `GET /api/v1/drift/` - List drift analyses
- `POST /api/v1/drift/` - Trigger new drift analysis
- `GET /api/v1/drift/{id}` - Get drift analysis details

### AI Insights
- `GET /api/v1/insights/` - List AI insights
- `POST /api/v1/insights/generate` - Generate new insights
- `GET /api/v1/insights/{id}` - Get insight details

### Prompt Templates
- `GET /api/v1/prompts/` - List prompt templates
- `PUT /api/v1/prompts/{id}` - Update prompt template
- `GET /api/v1/prompts/{id}/executions` - Get prompt execution logs

### Reports
- `GET /api/v1/reports/` - List reports
- `POST /api/v1/reports/generate` - Generate new report
- `GET /api/v1/reports/{id}/download` - Download report PDF

## Troubleshooting

### Backend Issues
- **bcrypt version error**: Ensure bcrypt is pinned to `<4.0.0` in requirements.txt
- **Database connection**: Check DATABASE_URL in .env file
- **OpenAI API**: Verify OPENAI_API_KEY is set correctly

### Frontend Issues
- **Build errors**: Run `npm run build` to check for TypeScript errors
- **API connection**: Ensure backend is running on http://127.0.0.1:8000
- **Authentication**: Use test credentials provided on login page
